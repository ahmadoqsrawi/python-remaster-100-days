# Python 100 Days

A personal 100-day challenge to build Python from the ground up, one day, one commit, one concept at a time.

## About

I used Python a while back but stepped away from it. After some time off, I decided to remaster it from the ground up, recap the fundamentals, and catch up with newer libraries. The goal is to sharpen my skills toward building scripts and tools for the security industry. To keep myself accountable, I made a 100-day challenge with a friend. Every day includes a Python task and a Git commit.

## Roadmap

| Phase | Days | Topic |
|-------|------|-------|
| 01 | 1–10 | Python Basics: setup, variables, data types, I/O, strings |
| 02 | 11–20 | Control Flow: if/else, loops, conditions |
| 03 | 21–30 | Functions: def, parameters, return values |
| 04 | 31–40 | Data Structures: lists, dicts, sets, tuples |
| 05 | 41–50 | Files & Errors: file handling, exceptions |
| 06 | 51–60 | OOP: classes, objects, inheritance |
| 07 | 61–70 | Modules: imports, packages, pip |
| 08 | 71–80 | APIs: requests, JSON, automation |
| 09 | 81–90 | Data: CSV, pandas |
| 10 | 91–100 | Projects: CLI tools, automation, final builds |
